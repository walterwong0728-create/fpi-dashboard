## 一、踩过的坑

### ❌ 4月17日：推荐流量突然归零

**现象：**
- 4月14-15日：推荐加购率4.3-4.7%
- 4月16日：推荐加购率1.9%（开始下滑）
- 4月17日：推荐加购率0%（彻底归零）

**当时错误做法：**
- 曾考虑直接暂停推荐流量

**正确认知：**
- 推荐流量是人群再营销的数据基础
- 宁可低效运转也不能关停
- 应该：更换人群包 + 优化素材

**经验：**
- 推荐渠道效果突然下降，先排查：人群包是否枯竭、商品位置是否被挤占
- 不要轻易暂停，要保留数据积累

---

### ❌ 4月16-17日：ROI持续低迷

**现象：**
- ROI连续2天低于3（16日2.39，17日2.62）
- 但消耗维持在¥120左右
- 访客量不低，但转化差

**原因分析：**
- 转化漏斗中间环节堵塞（加购骤降）
- 推荐渠道失效拖累了整体数据

**经验：**
- ROI低不一定是流量少，可能是转化漏斗问题
- 要分段排查：曝光→点击→加购→付款

**数据：**

**可复用策略：**
- 当各渠道均衡表现时，ROI最高
- 关键词+推荐双引擎模式

## 三、工作流程经验

### ✅ 数据更新标准流程

1. 接收Excel文件
2. 检查日期范围，确认是追加还是全量
3. 验证数据格式（字段对应）
4. 更新到GitHub
5. Vercel自动部署
6. 确认网站更新成功

### ❌ 踩过的流程坑

**问题：** 曾用minimaxi临时部署，导致网址每次都变

**解决：** 
- 固定用GitHub + Vercel
- GitHub push = 触发自动部署

**经验：** 
- 不要依赖临时部署
- 重要约定立刻存memory
